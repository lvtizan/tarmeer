# 邮件验证流程技术文档

本文档记录 Tarmeer 设计师平台的邮箱验证完整流程。

---

## 一、流程概览

```
用户注册 → 后端保存用户 → 发送验证邮件
    ↓
用户点击邮件中的 "Verify Email" 按钮
    ↓
前端调用验证接口 → 后端验证 token
    ↓
验证成功 → 返回 JWT 和用户信息
    ↓
前端存储登录状态 → 跳转设计师后台
```

---

## 二、详细流程

### 2.1 用户注册

**接口**: `POST /api/auth/register`

**请求体**:
```json
{
  "email": "designer@example.com",
  "password": "Password@123",
  "full_name": "John Doe",
  "phone": "+971501234567",
  "city": "Dubai"
}
```

**后端处理**:
1. 检查邮箱是否已注册
2. 检查是否为临时邮箱（tempmail 等）
3. 密码加密存储（bcrypt）
4. 生成验证 token（32位随机 hex）和过期时间（24小时）
5. 保存用户到数据库 `designers` 表
6. 发送验证邮件
7. 同时发送通知邮件给管理员

**数据库字段**:
| 字段 | 说明 |
|------|------|
| email | 用户邮箱 |
| password | 加密后的密码 |
| full_name | 用户姓名 |
| phone | 电话号码 |
| city | 城市 |
| verification_token | 验证 token |
| verification_expires | token 过期时间 |
| email_verified | 是否已验证（默认 false） |

---

### 2.2 发送验证邮件

**邮件服务**: 阿里云邮件推送 (DirectMail)

**配置** (`server/.env`):
```env
SMTP_HOST=smtpdm.aliyun.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=noreply@mail.kptom.com
SMTP_PASS=KPbec9f642d822488b
SMTP_FROM=noreply@mail.kptom.com
```

**邮件内容**:
- 收件人：用户注册邮箱
- 发件人：noreply@mail.kptom.com
- 主题：[Tarmeer] Verify Your Email Address
- 内容：包含用户姓名、Verify Email 按钮、有效期说明

**验证链接格式**:
```
{FRONTEND_URL}/verify-email?token={32位hex token}
```

本地开发: `http://localhost:5173/verify-email?token=xxx`  
线上环境: `https://designer.tarmeer.com/verify-email?token=xxx`

---

### 2.3 用户点击验证链接

**前端页面**: `/verify-email`（`src/pages/VerifyEmailPage.tsx`）

**处理流程**:
1. 从 URL 读取 token 参数
2. 显示加载动画
3. 自动调用验证接口
4. 根据结果显示成功/失败页面

```typescript
// VerifyEmailPage.tsx 核心逻辑
const token = searchParams.get('token');
const response = await api.post('/auth/verify-email', { token });

// 验证成功后存储
api.setToken(response.token);
localStorage.setItem('designer', JSON.stringify(response.designer));

// 跳转设计师后台
navigate('/designer/dashboard');
```

---

### 2.4 后端验证 Token

**接口**: `POST /api/auth/verify-email`

**请求体**:
```json
{
  "token": "abc123...（32位hex）"
}
```

**后端处理**:
1. 查询数据库找到匹配的 token
2. 检查 token 是否过期
3. 更新用户状态：`email_verified = true`
4. 清除验证 token
5. 生成 JWT（有效期 7 天）
6. 返回 JWT 和用户信息

**响应体**:
```json
{
  "message": "Email verified successfully.",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "designer": {
    "id": 1,
    "email": "designer@example.com",
    "full_name": "John Doe",
    "phone": "+971501234567",
    "city": "Dubai",
    "email_verified": true
  }
}
```

---

### 2.5 进入设计师后台

**状态管理**: `src/contexts/DesignerContext.tsx`

**加载用户信息**:
```typescript
// 从 localStorage 读取用户信息
const stored = localStorage.getItem('designer');
const data = JSON.parse(stored);

// 映射字段（处理后端命名差异）
{
  fullName: data.full_name || data.fullName,
  email: data.email,
  phone: data.phone,
  city: data.city,
  ...
}
```

**初始状态**:
- 用户信息：注册时填写的内容
- 项目列表：空数组（新用户）
- 提示用户完善个人资料和上传项目

## 四、数据库表结构

### designers 表

```sql
CREATE TABLE designers (
  id INT PRIMARY KEY AUTO_INCREMENT,
  email VARCHAR(255) NOT NULL UNIQUE,
  password VARCHAR(255) NOT NULL,
  full_name VARCHAR(100),
  phone VARCHAR(20),
  city VARCHAR(50),
  address VARCHAR(255),
  bio TEXT,
  avatar_url VARCHAR(500),
  title VARCHAR(100),
  email_verified BOOLEAN DEFAULT FALSE,
  verification_token VARCHAR(64),
  verification_expires DATETIME,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## 五、错误处理

### 5.1 注册错误

| 错误 | 原因 | 处理 |
|------|------|------|
| Email already registered | 邮箱已被注册 | 提示用户直接登录 |
| Temporary email addresses are not allowed | 使用临时邮箱 | 提示使用真实邮箱 |

### 5.2 验证错误

| 错误 | 原因 | 处理 |
|------|------|------|
| Invalid verification link | token 无效 | 提示重新注册或联系支持 |
| Verification link has expired | token 过期（24小时） | 提供"重新发送验证邮件"功能 |

### 5.3 登录错误

| 错误 | 原因 | 处理 |
|------|------|------|
| Please verify your email address first | 邮箱未验证 | 显示"重新发送验证邮件"按钮 |
| Invalid email or password | 邮箱或密码错误 | 提示检查输入 |

---

## 六、安全措施

1. **密码加密**: 使用 bcrypt，salt rounds = 10
2. **Token 安全**: 32 位随机 hex，24 小时过期
3. **JWT 有效期**: 7 天
4. **HTTPS**: 线上强制 HTTPS
5. **临时邮箱过滤**: 禁止常见临时邮箱域名注册
