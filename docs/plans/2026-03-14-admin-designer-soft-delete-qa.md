# Admin Designer Soft Delete QA Checklist

## 测试目标

验证后台“删除设计师/恢复设计师”功能在本地环境可用，并确保公开站不会继续展示已删除设计师。

## 本地环境

- 前端：`http://127.0.0.1:5173`
- 后台登录页：`http://127.0.0.1:5173/admin/login`
- 设计师管理页：`http://127.0.0.1:5173/admin/designers`

## 必测场景

1. 删除一个 `approved` 设计师
- 在列表页点击 `Delete`
- 不填写理由时不能提交
- 填写理由后删除成功
- 设计师从 `Active` 视图消失
- 在 `Deleted` 视图可见

2. 恢复一个已删除设计师
- 在 `Deleted` 视图点击 `Restore`
- 恢复后设计师重新出现在 `Active` 视图
- 删除理由与删除时间不再显示为当前有效状态

3. 详情页删除/恢复
- 从详情页执行 `Delete`
- 删除成功后详情页显示 `Designer deleted`
- 点击 `Restore` 后恢复正常

4. 已删除设计师不再公开展示
- `http://127.0.0.1:5173/designers` 中不再显示该设计师
- 直接打开原设计师公开详情页应不可正常访问
- 该设计师项目不应继续在公开接口/公开详情中可见

5. 已删除设计师不应继续使用旧会话
- 若该设计师已有登录态，删除后再执行受保护操作应被拒绝

6. 不影响其他后台能力
- 其他设计师审批功能仍正常
- 项目审批功能仍正常
- 公开设计师页仍正常加载图片与详情弹层

## 通过标准

- 所有场景通过
- 无控制台致命错误
- 无公开页回归
- 无误删后无法恢复的问题

## 若发现问题

记录：
- 测试设计师名称
- 操作步骤
- 实际结果
- 预期结果
- 截图/录屏
