# Edit Project Redesign Design

日期：2026-03-16

## 背景与目标

本次改版聚焦设计师提交项目体验，保留现有核心业务能力（上传、排序、设封面、草稿、提交），解决四类问题：

1. 顶部流程信息重复，抢占首屏注意力。
2. 图片管理区权重不足，封面反馈与操作清晰度不够。
3. 表单偏后台填表感，提交状态感知弱。
4. 提交后等待感强，用户感知“项目不见/未完成”的焦虑高。

## 设计原则

1. 少解释，多反馈。
2. 弱化重复信息，只保留一个状态源。
3. 图片优先，但保持页面双列网站布局不变。
4. 可提交性可见：始终明确“能否提交”和“缺什么”。
5. 提交体验无感化：先看到项目卡片，再自动完成上传状态切换。

## 交互与结构设计

### 1) 顶部区域

- 保留：返回入口、页面标题、`Save Draft`、`Submit for Review`。
- 删除：`Upload Flow`、大进度条、步骤 chips 等重复流程表达。
- 新增：简洁状态栏，仅显示：
  - `Ready to submit`
  - `Incomplete submission` + `Missing: ...`

### 2) 左侧 Project Details

- 维持必要字段：标题、描述、风格、城市、面积、年份。
- `Description` 从强必填改为推荐填写。
- 元数据采用紧凑分组布局（Style/City + Area/Year）。
- 增加自动保存状态文案：`Saving... / Saved just now / Failed to save`。

### 3) 右侧 Image Board（核心）

- 保持两列总布局不变，右侧为紧凑型管理面板。
- 上传区支持拖拽/点击，拖入高亮，错误原因可见。
- 顶部状态显示：图片数量、封面状态、总大小。
- 图片卡支持：预览、设封面、删除（二次确认）、拖拽排序。
- 封面反馈强化：卡片高亮 + `Cover` 角标 + 顶部 `Current Cover` 预览。
- 图库区采用紧凑网格 + 内部滚动，尽量单屏看全。

### 4) 提交与“无感上传”

- 创建项目提交时，先在 `My Projects` 列表中插入 `uploading...` 卡片。
- 后台请求成功后自动替换为真实项目数据并显示封面。
- 请求失败时自动重试（指数退避，最多 8 次），尽量无感恢复。
- 超过重试上限才进入失败态并允许人工重试。

## 状态规则

### 可提交必填

- Project Title
- Style
- Location (City)
- 至少 1 张图片
- 已选 Cover（通过 `coverIndex`）

### 状态反馈

- 顶部状态栏作为唯一主反馈源。
- `Submit` 不可用时，显示具体缺失字段组合。
- 图片上传与排序变化即时反馈，避免弱提示。

## 验收标准映射

1. 顶部冗余流程块删除，首屏聚焦“编辑+提交”。
2. 两列布局保持，右侧图片区成为高效管理区。
3. 不可提交时明确缺项，不再仅灰按钮。
4. 封面识别明显，能快速确认当前封面。
5. `My Projects` 可见 `uploading...` 过渡态并自动切换完成态。
6. 删除使用二次确认，减少误删。
