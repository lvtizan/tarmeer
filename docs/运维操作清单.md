# Tarmeer 4.0 — 运维操作清单（可打印）

**服务器**: 47.91.108.104 | **后端端口**: 3002 | **后端目录**: /tarmeer/tarmeer_api

---

## 一、Nginx 配置（必做）

1. **打开项目中的配置参考文件**  
   `docs/nginx-api-for-ops.conf`

2. **在现有站点的 server { } 内增加下面这段**（与主站共用 80 时，只加 location 即可）：oad

```nginx
location /api {
    proxy_pass http://127.0.0.1:3002;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

3. **检查配置**  
   ```bash
   nginx -t
   ```

4. **重载 Nginx**  
   ```bash
   systemctl reload nginx
   ```  
   （若你们用其他方式重载，按现有流程执行。）

---

## 二、验证（建议做）

在服务器或本机执行：

```bash
curl http://47.91.108.104/api/health
```

返回 JSON（如 `{"status":"ok",...}`）即表示 API 与 Nginx 正常。

---

## 说明

- 本仓库与部署脚本**不会自动修改** Nginx，以上配置需运维**人工合并**并重载。
- 后端服务需已部署并监听 `127.0.0.1:3002`（由开发/后端同事部署）；若未部署，请先完成后端部署后再做 Nginx 配置。
