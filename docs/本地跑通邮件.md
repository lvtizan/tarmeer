# 本地跑通验证邮件（数据存本地库，再部署 ECS）

## 一、当前配置（server/.env）

- **数据库**：**本地 MySQL**（localhost），测试数据不写 RDS，无需找后台删账号  
- **SMTP**：阿里云邮件推送（noreply@mail.kptom.com），与公司一致  
- **验证链接**：FRONTEND_URL=http://localhost:5173  
- **前端端口**：5173（vite.config.ts 已设为 5173，访问 http://localhost:5173/auth ）

## 二、本地 MySQL 建库（首次必做）

本机需已安装 MySQL，在项目目录执行：

```bash
mysql -u root -p < server/schema/init.sql
```

按提示输入 root 密码。会创建数据库 `tarmeer` 及表 `designers`、`projects`、`contacts`。

若 root 无密码，可：

```bash
mysql -u root < server/schema/init.sql
```

**server/.env 里**：`DB_USER=root`，`DB_PASSWORD=`（无密码时留空）；若有密码则填 `DB_PASSWORD=你的密码`。

## 三、本地运行步骤

### 1. 启动后端

```bash
cd server
npm install
npm run dev
```

出现 `Server running on port 3002` 即成功。

### 2. 启动前端（新开终端）

```bash
# 项目根目录
npm install
npm run dev
```

浏览器打开 **http://localhost:5173** ，注册页：**http://localhost:5173/auth** 。

### 3. 测试收邮件

1. 打开 http://localhost:5173/auth ，切到「Create Account」  
2. 用真实邮箱注册并提交  
3. 收件箱（含垃圾箱）查收来自 noreply@mail.kptom.com 的验证邮件  
4. 点击邮件里的「Verify Email」，会打开 http://localhost:5173/verify-email?token=... ，验证成功后即可登录  

测试数据都在本地 MySQL 的 `tarmeer` 库，可随时清空或重建（重跑 init.sql 或手动删表数据），不影响线上。

## 四、若 http://localhost:5173/auth 无法打开

- 确认前端已执行 **npm run dev**（项目根目录），且终端无报错  
- 确认访问地址为 **http://localhost:5173/auth**（端口 5173，路径 /auth）  
- 若 5173 被占用，终端会提示改用其他端口，按提示访问即可  

## 五、本地跑通后再部署 ECS

1. 部署后端：`./deploy-backend-ecs.sh`  
2. ECS 上 `/tarmeer/tarmeer_api/.env` 使用 RDS + 阿里云 SMTP（见 docs/SMTP验证邮件配置.md），执行 `pm2 restart tarmeer-api`  
3. 前端：`./deploy-ecs.sh`  

线上注册仍用 RDS + 阿里云 SMTP，与本地测试互不影响。
