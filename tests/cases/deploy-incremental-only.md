# 生产部署仅增量 rsync - 测试用例

## TC-DP-001 脚本仅走增量同步

- 前置条件：
  - 具备服务器部署权限
  - 已阅读部署规则文档
- 步骤：
  1. 执行 `DEPLOY_RULES_ACK=YES bash deploy-simple.sh`
  2. 观察步骤输出
- 预期结果：
  - 输出包含 `增量同步文件到服务器`
  - 使用 `rsync` 传输（非 `rm -rf` 清空远端）
  - 部署成功结束并打印站点地址

## TC-DP-002 无变更时近似零增量

- 前置条件：
  - 已完成一次成功部署
  - 本地代码无新改动
- 步骤：
  1. 再次执行 `DEPLOY_RULES_ACK=YES bash deploy-simple.sh`
  2. 查看 `rsync` 统计信息
- 预期结果：
  - 仅少量或无文件变化
  - 同步统计显示传输量明显低于首次部署

## TC-DP-003 权限与 Nginx 重载验证

- 前置条件：
  - 脚本执行到权限步骤
- 步骤：
  1. 观察 `nginx -t` 输出
  2. 观察健康检查输出
- 预期结果：
  - `nginx` 配置检测通过
  - 首页检查返回 `200`
  - 头像图片检查返回 `200`

## TC-DP-004 构建产物缺失保护

- 前置条件：
  - 临时将本地 `dist/` 重命名或删除
- 步骤：
  1. 执行 `DEPLOY_RULES_ACK=YES bash deploy-simple.sh`
- 预期结果：
  - 脚本在增量同步前中断
  - 输出 `本地目录不存在` 相关错误
